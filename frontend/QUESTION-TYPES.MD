# Типы вопросов + промпт для конструктора опросов

> Файл для Claude Code. Список опросов (Глубокий, Пульс, 1:1, Новый 30/90
> дней) уже реализован в системе — этот файл НЕ про него. Здесь только:
> (1) какие типы вопросов должен поддерживать конструктор внутри опроса,
> и (2) промпт, чтобы Claude Code сделал сам конструктор (блоки + вопросы,
> drag&drop).

---

## Типы вопросов

| # | Тип | Описание | Пример из наших опросов |
|---|-----|----------|--------------------------|
| 1 | **NPS-шкала (0–10 / 1–10)** | Горизонтальная шкала с подписями по краям | "Порекомендовали бы Decor Center другу?" |
| 2 | **Шкала оценки 1–5** | То же самое, но 5 делений (Лайкерт) | "Удовлетворены ли зарплатой?" |
| 3 | **Один вариант ответа (radio)** | Выбор одного из 3-4 вариантов | "Планируете ли остаться в компании?" |
| 4 | **Несколько вариантов (checkbox)** | Выбор нескольких из списка (до 8) | "Что мотивирует Вас? (несколько)" |
| 5 | **Короткий текст** | Ответ в одну строку | "Одно предложение на следующий квартал" |
| 6 | **Длинный текст / MIND DIVE** | Развёрнутый открытый ответ | "Что бы Вы изменили в компании и почему?" |
| 7 | **Поле формы (текст/дата)** | Реквизиты: ФИО, должность, дата, стаж | Блок "Общая информация" |
| 8 | **Подпись + дата** | Специальный блок в конце опроса | "Подпись сотрудника ___ Дата ___" |
| 9 | **Заголовок блока/раздела** | Не вопрос, а разделитель секций | "II. ЛОЯЛЬНОСТЬ" |

Дополнительно — на будущее (заложить в архитектуру, но не обязательно делать в первой версии):

| # | Тип | Описание |
|---|-----|----------|
| 10 | Выпадающий список (dropdown) | Один выбор из длинного списка |
| 11 | Дата (date picker) | Отдельное поле-календарь |
| 12 | Число (number) | Числовое поле с min/max |
| 13 | Матрица/сетка | Несколько строк с одной общей шкалой |
| 14 | Ранжирование (ranking) | Расставить варианты по порядку |
| 15 | Загрузка файла | Прикрепление файла к ответу |

### Общие свойства для любого типа вопроса
- Текст вопроса на **двух языках** (uz + ru) одновременно.
- Флаг **"обязательный"**.
- Флаг **"MIND DIVE"** (открытый глубокий вопрос — для аналитики).
- Порядковый номер (`order`) — для drag&drop.
- Привязка к блоку (`block_id`).

### Специфичные настройки по типам
- **Шкалы (NPS/1-5)**: min, max, подпись левого края (uz/ru), подпись правого края (uz/ru).
- **Radio/Checkbox/Dropdown**: список вариантов (каждый — uz/ru), опция "свой вариант".
- **Текст**: плейсхолдер (uz/ru), максимальная длина (опционально).
- **Матрица**: список строк (uz/ru) + общая шкала/варианты.
- **Число**: min, max.

## Промпт для Claude Code

```
Нужно сделать конструктор внутри опроса — блоки и вопросы с drag&drop,
чтобы при заходе в опрос можно было сразу добавлять блоки, добавлять внутрь
них вопросы и менять их порядок перетаскиванием.

Сначала проверь бэкенд:
1. Как сейчас устроена структура опроса в БД/коде (Survey, Block/Section,
   Question)? Поддерживает ли она порядок (order) для блоков и вопросов?
2. Поддерживает ли схема двуязычные тексты (uz/ru) для вопросов и вариантов
   ответа?
3. Есть ли уже enum/список типов вопросов, и какие типы туда входят?

Если бэкенд уже поддерживает нужную структуру — просто добавь недостающие
типы вопросов и endpoints для reorder (изменение порядка при drag&drop).
Если не поддерживает — спроектируй и добавь новую схему/миграции и API.

Типы вопросов, которые обязательно должны поддерживаться:
1. NPS-шкала (0-10 или 1-10) с подписями по краям (uz/ru)
2. Шкала оценки 1-5 с подписями по краям (uz/ru)
3. Один вариант ответа (radio)
4. Несколько вариантов ответа (checkbox)
5. Короткий текстовый ответ
6. Длинный текстовый ответ / открытый вопрос (MIND DIVE)
7. Поле формы (текст или дата) — для реквизитов типа ФИО, должность, дата
8. Блок "Подпись + дата" в конце опроса
9. Заголовок блока/раздела (не вопрос, а разделитель)

Заложи в архитектуру (можно не реализовывать сразу, но не блокировать
на будущее): выпадающий список, дата, число, матрица/сетка, ранжирование,
загрузка файла.

У каждого вопроса: текст на uz и ru, флаг "обязательный", флаг "MIND DIVE",
порядок (order), привязка к блоку.

Фронтенд: drag&drop для блоков целиком и для вопросов внутри блока и между
блоками (для React — используй @dnd-kit/core + @dnd-kit/sortable). При
добавлении вопроса — выпадающее меню с выбором типа. У каждого вопроса —
панель редактирования текста (uz/ru), настроек типа, обязательности,
кнопки удалить/дублировать/переместить. Автосохранение при любом изменении.

Не ломай существующую функциональность и данные опросов. Сначала покажи мне
план (какие файлы/таблицы/эндпоинты будешь менять), потом реализуй по шагам.
```
