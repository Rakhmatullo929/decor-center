---

## 1. PROJECT OVERVIEW

1.1. The purpose of the system is to regularly assess the professional knowledge and occupational-safety knowledge of depot employees, and to record the results of daily medical examinations in a single electronic system.

1.2. The system consists of three core modules:

1. **Professional knowledge assessment** — tests generated by AI based on official instructions; questions are read aloud (TTS); identity is confirmed via Face ID.
2. **Occupational-safety knowledge assessment** — tests in the areas of technical safety and industrial safety; questions are text-only (no audio).
3. **Medical examination** — recording of arterial blood pressure, pulse, oxygen saturation, breathalyzer results, and work/rest hours, followed by a conclusion.

1.3. The system runs in a browser and must operate on a computer with a connected camera (required for Face ID).

## 3. USER ROLES AND ACCOUNTS

3.1. The system has three types of accounts:

| # | Account | Quantity | Purpose |
|---|---|---|---|
| 1 | Specialists account | 1 (shared) | All employees use this account to take tests |
| 2 | Administrator | 1 | Manages employees, tests, instructions, and results |
| 3 | Medical staff | 1 | Enters daily medical examination data |

3.2. Permissions:

- **Specialists account** — test-taking flow only; result visibility is limited to the post-test screen; cannot modify anything.
- **Administrator** — full control: employees, specialties, instructions, tests, results, reports, accounts.
- **Medical staff** — entering medical examination records and viewing their history.

3.3. Authorization is performed via login/password. Passwords are stored hashed, and role-based permissions are verified on the server side.

## 4. EMPLOYEE DATABASE

4.1. The employee database is maintained by the administrator. Each employee record contains the following fields:

| Field | Description | Required |
|---|---|---|
| Full name | Complete surname, first name, patronymic | Yes |
| Specialty | Selected from a list | Yes |
| Photo | Reference photo for Face ID (frontal, clear) | Yes |
| Status | Active / Inactive | Yes |

4.2. List of specialties (extendable by the administrator):

1. Engine driver (machinist)
2. Assistant engine driver
3. Depot duty officer
4. Locksmith (fitter)
5. Engine driver for training affairs
6. Chief master for training affairs

4.3. When a photo is uploaded, the system generates and stores a face embedding (digital signature) from it — Face ID comparison is based on this. If no face is detected in the photo, the system warns the user and requests another photo.

## 5. MODULE 1 — PROFESSIONAL KNOWLEDGE ASSESSMENT

### 5.1. Building the test bank

5.1.1. For each specialty, the administrator uploads the depot's official instructions (PDF, DOCX, or text) into the system.

5.1.2. The AI automatically generates tests based on the uploaded instructions. Each test item consists of:

- question text;
- 4 answer options;
- 1 correct answer;
- the specialty it belongs to.

The number of questions generated per run is configurable.

5.1.3. AI-generated tests are initially in a "Not approved" state. The administrator reviews, edits, approves, or deletes them. Only approved tests are shown to employees.

5.1.4. The administrator can also add, edit, and delete tests manually.

### 5.2. Test-taking process

1. Log in to the specialists account.
2. The employee selects their full name from the list (with search).
3. **Face ID check:** the employee looks at the camera, and the system compares the live image with the reference photo in the database.
4. If the face does not match or is not detected, the test **does not start**, a warning appears on screen, and the attempt is logged.
5. If the face is confirmed, the system randomly selects **10 questions** from the approved tests belonging to the employee's specialty.
6. Each question is displayed on screen and **read aloud via AI voice (TTS)**. A "Replay" button is provided.
7. The employee selects one answer per question and moves to the next.
8. At the end of the test, the result is calculated automatically and saved to the database.

### 5.3. Stored result data

- employee and their specialty;
- start and end date/time;
- presented questions, given answers, correct/incorrect flag;
- total score (e.g., 8 out of 10 correct);
- the fact that Face ID was passed successfully;
- pass/fail status (passing score to be agreed in Section 15).

---

## 6. MODULE 2 — OCCUPATIONAL-SAFETY KNOWLEDGE ASSESSMENT

6.1. The module consists of two areas:

1. **Technical safety** tests;
2. **Industrial safety** tests.

6.2. The test bank is built **manually through the administrator panel only**: question text, 4 options, 1 correct answer, area.

6.3. The test-taking process is the same as in Section 5.2, with the following differences:

- the employee first selects an area (technical safety or industrial safety);
- questions are **not voiced** — they are shown as text only;
- number of questions — 10 (randomly selected).

6.4. The Face ID check is identical to Module 1: if the face is not confirmed, the test does not start.

6.5. Results are saved as in Section 5.3, with the area indicated.

---

## 7. MODULE 3 — MEDICAL EXAMINATION

7.1. Through the medical staff account, the following data is entered daily for each employee:

| Field | Description |
|---|---|
| Employee | Selected from a list (search by full name) |
| Date and time | Set automatically |
| Arterial blood pressure (BP) | Systolic / diastolic (e.g., 120/80) |
| Pulse | Beats per minute |
| Oxygen saturation (SpO₂) | In percent |
| Breathalyzer result | Value, or negative/positive (format to be agreed in Section 15) |
| Hours worked | Number of hours worked in the last shift |
| Hours rested | Number of hours rested after the shift |
| Conclusion | **Approved / Rejected** |
| Note | Optional text field |

7.2. The system automatically records who entered each record and when.

7.3. Editing a saved conclusion afterwards is prohibited (or allowed only with administrator permission — to be agreed in Section 15); a full change history is retained for audit purposes.

7.4. History and search: filtering by employee, by date range, and by conclusion; list and detail views.

7.5. **Recommendation:** if indicators are out of normal range (e.g., a positive breathalyzer result), the form provides a visual warning; the final decision is always made by the medical staff.

## 11. NON-FUNCTIONAL REQUIREMENTS

11.1. **Interface language** — Uzbek (Latin). Russian may be added later if required.

11.2. **Security:**

- HTTPS is mandatory (the getUserMedia camera function works only over HTTPS or on localhost);
- role-based permissions are verified on the server side;
- passwords are hashed; session lifetime is limited;
- Face ID attempts (successful and failed) are logged.

11.3. **Browsers:** the latest versions of Chrome and Edge.

11.4. **Interface:** primarily for desktop (camera workstation), with a responsive design.

11.5. **Reliability:** daily automatic backup (PostgreSQL dump); the retention period for results is agreed with the client.

11.6. **Performance:** at least 20–30 concurrent users; pages load in under 2 seconds.

---

## 12. EXTERNAL SERVICES AND INTEGRATIONS

12.1. **AI test generation.** The instruction text is split into chunks, and questions are generated from each chunk via the LLM API. The result is saved into the system in a "Not approved" state and is used only after the administrator approves it.

12.2. **TTS (voice-over).** A service is selected for reading Module 1 questions aloud. Options for language support are presented to and agreed with the client.

12.3. **Face recognition.** Performed server-side: an embedding is taken from the reference photo and compared with the frame from the camera. The similarity threshold is configurable. Recommendation: protection against spoofing with a photo (liveness check) is agreed separately.

## 14. ACCEPTANCE CRITERIA

The system is considered accepted when the following conditions are met:

1. All three account types work, with permissions correctly restricted.
2. A test starts only after the employee is selected from the list and passes the Face ID check; a test will not open for an unrecognized face.
3. In Module 1, 10 random questions are presented, questions are read via TTS, and the result is saved.
4. In Module 2, tests work for both areas (without audio).
5. The AI generates tests from instructions, and the administrator approves them; the administrator can add and delete tests manually.
6. The medical examination form accepts all indicators and saves with an "Approved / Rejected" conclusion; history can be viewed.
7. Results work with filtering and Excel export.
